Handoff moves application state between devices without requiring an
account or cloud sync.

You have state here. You want it there. The industry answer is an account,
a server in another country, a database row, and a sync engine — to move
something across the room, once. Handoff is the smaller primitive: the
state becomes a self-contained artifact that travels as a QR code or a
link, and the receiving device becomes the sending device, exactly.

Handoff is not Dropbox, AirDrop, cloud sync, or a collaboration backend.
It is not even "sync without accounts" — that would be a much larger
product. [Transfer is not sync](/handoff/docs/concepts/transfer-vs-sync),
and Handoff only transfers.

## Used in Local

<Callout type="note">
[Local](https://local.sarwagya.wtf) uses Handoff to move the WebRTC
offer/answer bootstrap between two browsers as a QR code or a link — the
connection state only, never the messages. No signaling server in the
middle. Source:
[github.com/0xsarwagya/local](https://github.com/0xsarwagya/local).
</Callout>

## Status

<Callout type="warning">
Handoff is experimental. The v1 wire protocol is versioned and pinned by
test vectors, but the API may still change before 1.0. Inline transfer is
the only transport — by design, until it earns a second one.
</Callout>
